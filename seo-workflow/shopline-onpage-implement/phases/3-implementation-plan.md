# Phase 3 — Implementation Plan

Cross-reference audit findings (Phase 1) with the Shopline snapshot (Phase 2b),
GSC/GA4 data (Phase 2a), and market signals (Phase 2c — SOCIAL-TRENDS) to produce
a clear, categorised implementation plan. This is what the user approves in Phase 4.

**Load before starting:**
- `audit/AUDIT-YYYY-MM-DD.md` — SEO findings
- `research/GSC-REPORT-YYYY-MM-DD.md` and/or `GA4-REPORT-YYYY-MM-DD.md` — performance gaps
- `research/SOCIAL-TRENDS-YYYY-MM-DD.md` — what the audience is talking about right now
- `implementation/SHOPLINE-SNAPSHOT-YYYY-MM-DD.md` — current store state

## Goal

A document that shows:
- Exactly what will change (before → after for every field)
- Which category each change falls into
- What CANNOT be done via API (manual work flagged)
- A risk note for any change that touches live-indexed pages

---

## Categories

Only include categories that have actual changes.

---

### Category A — Page SEO Titles

For every custom page where `seoTitle` is missing or suboptimal:

| Page ID | Page | Slug | Current seoTitle | Proposed seoTitle | Reason |
|---|---|---|---|---|---|
| ghi789 | About Us | /about-us | (missing) | About Our Team — Brand | Keyword-rich, descriptive |

**Title rules:**
- Lead with primary keyword, brand name at end separated by em dash (—)
- 50–60 characters
- Never use store name acronyms or truncated handles as the title

---

### Category B — Page Meta Descriptions

For every page where `seoDescription` is missing or auto-generated:

| Page ID | Page | Slug | Current seoDescription | Proposed seoDescription |
|---|---|---|---|---|

### Category F — Schema Injection (via blog post body or page HTML)

Shopline does not have a script injection API like Webflow. Schema is added by:
1. Embedding JSON-LD directly in the `content_html` of blog posts (for Article schema)
2. Requesting theme-level injection via the Theme Editor (manual — flag in Category G)

For blog posts: include the JSON-LD `<script>` block at the top of `content_html` where appropriate.

| Article ID | Schema Type | Why |
|---|---|---|
| abc123 | Article + BreadcrumbList | E-E-A-T signal, AI citation readiness |

---

### Category G — Cannot Be Done Via API (Manual Work Required)

Always include this section. Users need to know what the API cannot touch.

| Item | Why It Can't Be Done | Manual Action Required |
|---|---|---|
| Organization schema (homepage) | Requires Theme Editor to inject into `<head>` | Add JSON-LD script to theme `theme.liquid` |
| FAQ schema | Requires Theme Editor or page template | Add to page template or page content |
| H1/H2 headings on pages | Shopline page templates control headings | Edit in Theme Editor |
| robots.txt | Store-level setting | Set in Shopline Admin > Settings > SEO |
| Sitemap | Auto-generated by Shopline | Configure in Admin > Settings > SEO |
| URL slug changes on live pages | Risks breaking indexed URLs + rankings | Plan redirect strategy first |
| Navigation menu changes | Admin only | Shopline Admin > Online Store > Navigation |
| Image alt text on theme assets | Theme Editor only | Edit images in Theme Editor |
