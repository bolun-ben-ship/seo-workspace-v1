## Change Log 3 — Schema / Structured Data

Four JSON-LD scripts were registered in Webflow and applied to their target pages. This takes the site from **zero schema** to full coverage across business entity, FAQs, and blog posts.

### Script 1: Organisation + FinancialService Schema
**Applied to:** Homepage (`/`)
**Purpose:** Establishes AEXPHL as a named entity in Google's Knowledge Graph. Enables rich results for business panels, logo display in SERPs, and cross-platform entity recognition (LinkedIn, Facebook sameAs links included).

**Before:** No schema detected
**After:**
```json
{
  "@context": "https://schema.org",
  "@type": ["Organization", "FinancialService"],
  "name": "Aussie Expat Home Loans",
  "alternateName": "AEXPHL",
  "url": "https://www.aexphl.com",
  "logo": "https://cdn.prod.website-files.com/5e1ef6e0ff2a7e7d638dd146/5e1ef6e0ff2a7e03f08dd148_logo.svg",
  "description": "Specialist Australian mortgage brokerage for expats in Singapore, Hong Kong and Dubai.",
  "areaServed": ["AU", "SG", "HK", "AE"],
  "sameAs": ["https://sg.linkedin.com/company/aexphl", "https://www.facebook.com/Aussieexpathomeloans/"],
```

### Script 3: Article + BreadcrumbList Schema
**Applied to:** Blog Posts Template (all blog posts dynamically)
**Purpose:** Marks all blog posts as structured Articles with publisher info. BreadcrumbList schema improves SERP display for blog URLs. Both signals improve E-E-A-T and AI citability of blog content.

**Before:** No schema on any blog posts
**After:** Dynamic Article schema (headline from `document.title`, URL from `window.location.href`) + BreadcrumbList with Home → Blog → Post path on every blog post.

---

### Script 4: NoIndex Tag (Robots Control)
**Applied to:** 19 non-production pages (see full list in Change Log 4)
**Purpose:** Prevents Google from indexing ad landing pages, staging/v2 pages, thank-you pages, and internal utility pages that dilute the site's SEO equity.

**Implementation note:** The noindex is injected via JavaScript (`document.head.insertAdjacentHTML`). This means:
- It takes effect on pages where the script is applied
- It **overrides** Webflow's native "index" toggle — Google respects the most restrictive robots signal
- To re-index any of these pages in future, the script must be removed from that page via the API **in addition to** toggling Webflow's index setting

## Noindex Management Reference

> **Important:** The noindex scripts injected via the Webflow custom code API **override** Webflow's native "index" toggle. The script system is the source of truth for noindex on affected pages.

To re-index any noindexed page in future, two steps are required:
1. Remove the `noindextag` script from that specific page via the Webflow API (Claude can do this on request)
2. Confirm Webflow's native index setting is ON for that page
