- `/landing-page-v2` appears to be a staging/revamp version. If live traffic is not intended, this should be noindexed or 301 redirected to the canonical homepage.

### Canonicalisation
With two near-identical homepage variants indexed (`/` and `/landing-page-v2`), there is a canonicalisation risk. Both may split link equity and create duplicate content signals.

### Redirects
The domain appears to redirect correctly from `http://` and `aexphl.com` to `https://www.aexphl.com/`. No 3xx chain issues detected from SERP data.

### Site Architecture

## Schema / Structured Data

**Score: 18/100**

No schema markup detected in SERP results (no rich snippets appearing for any pages). For a YMYL financial services site, this is a significant gap.

### Missing Schema — High Priority
| Schema Type | Page | Value |
|---|---|---|
| `Organization` | Homepage | Brand entity establishment, logo, sameAs (LinkedIn, Facebook) |
| `LocalBusiness` (Financial Service) | Homepage | Maps/local pack eligibility, multi-location support |
| `FAQPage` | `/faqs` | Direct rich result in SERP — immediate CTR lift |
| `BreadcrumbList` | All pages | Cleaner SERP display, site architecture signal |
| `Article` + `Author` | All blog posts | E-E-A-T signal for financial content, AI citation readiness |
| `Service` | `/services` | Structured service listing, eligible for Knowledge Panel |

### Implementation Note
Webflow allows custom `<script type="application/ld+json">` embeds in page settings or via custom code components. This is not a platform limitation — it requires implementation.

## AI Search Readiness (GEO)

**Score: 28/100**

AI search engines (ChatGPT, Perplexity, Gemini) increasingly cite financial services content in answer panels. AEXPHL is not currently positioned for this.

### Why AEXPHL Should Care
The target user — an Aussie expat in Singapore thinking about buying property — is exactly the type of person who asks AI assistants: *"Can I get a mortgage in Australia while living in Singapore?"* or *"How does foreign income affect Australian home loan applications?"*

### Current Gaps
- No Article schema → blog posts aren't structured for AI consumption
- No Author schema → no E-E-A-T signal for AI citation selection
- Blog posts lack clear, quotable definitions or structured Q&A sections that AI prefers to cite
- No FAQ schema → FAQs are invisible to AI overview features
- No explicit "expert" positioning copy that AI systems use to identify authoritative sources

### Quick Wins for GEO
- Add structured FAQ blocks within blog posts (then schema-mark them)
- Add author profiles with credentials (Tim, Shaun, Charu) on all blog posts
- Reframe 2-3 core blog posts with clear definitive answers in the first 150 words (AI cites these)
